# 专利业务模型设计

> 核心原则：以专利为锚点，全生命周期、全维度、跨国界
> 更新日期：2026-07-19

## 一、业务定位与核心场景

### 1.1 产品定位

**跨国专利风险管理与专利申请管理的综合平台**——专利部门的核心工作系统。

**不是什么**：
- 不是专利检索工具（但可以导入检索结果）
- 不是专利分析工具（但可以做基础分析）
- 不是文档管理系统（但管理专利相关文档）

**是什么**：
- 专利资产的**中央数据库**
- 专利全生命周期的**流程管理系统**
- 专利风险的**预警与跟踪系统**
- 跨部门协作的**信息枢纽**
- 专利知识的**沉淀与传承平台**

### 1.2 核心使用场景

| 场景 | 主要角色 | 核心需求 |
|------|---------|---------|
| **专利申请管理** | 专利工程师、代理人 | 从提案到授权的全流程跟踪、期限管理、文档管理 |
| **专利风险管理** | 法务、产品、研发 | 竞品专利监控、风险评估、应对措施跟踪 |
| **专利布局管理** | 专利工程师、产品经理 | 按产品线/项目/技术领域的专利布局 |
| **年费与成本管理** | 专利工程师、财务 | 年费提醒、成本分析、预算管理 |
| **技术参考管理** | 研发、产品 | 技术调研、参考专利收集、技术地图 |
| **无效/诉讼管理** | 法务、外部律师 | 无效案件、诉讼案件的文件与进度管理 |
| **许可/转让管理** | 法务、商务 | 许可合同、转让交易的管理 |
| **部门看板与报表** | 管理层、部门负责人 | 数据统计、趋势分析、决策支持 |

### 1.3 核心用户角色

| 角色 | 系统使用深度 | 核心功能 |
|------|-------------|---------|
| **专利工程师/代理人** | 深（每天用） | 全功能，日常操作主力 |
| **研发人员** | 浅（偶尔用） | 提交发明提案、查看相关专利、提供技术支持 |
| **产品/项目经理** | 中（每周用） | 项目维度专利视图、风险跟踪、布局规划 |
| **法务/合规** | 中（按需） | 风险评估、合同审核、诉讼支持 |
| **部门负责人** | 中（每周/每月） | 看板、报表、审批 |
| **管理层** | 浅（每月/每季度） | 高层视图、战略数据 |
| **外部律师/代理机构** | 受限（按需） | 仅能看到委托的案件，协作功能 |

## 二、专利全生命周期模型

### 2.1 生命周期阶段定义

```
┌─────────────────────────────────────────────────────────────────┐
│                      专利全生命周期                               │
├─────────┬──────────┬──────────┬──────────┬──────────┬──────────┤
│  提案期  │  申请期   │  审查期   │  授权期   │  维护期   │  终态    │
└─────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

### 2.2 各阶段详细说明

#### 阶段1：提案期（内部）

| 子阶段 | 说明 | 关键产出 | 责任人 |
|--------|------|---------|--------|
| 创意产生 | 研发/产品有了新想法 | 创意记录 | 研发 |
| 发明提案 | 正式提交发明提案 | 发明提案表 | 研发 |
| 内部评审 | 专利性、商业价值评估 | 评审意见 | 专利工程师 + 评审组 |
| 立项决策 | 决定是否申请 | 决策记录 | 部门负责人 |
| 交底书撰写 | 详细技术交底 | 技术交底书 | 研发 + 专利工程师 |

**核心字段**：
- 提案编号、提案日期、提案人、所属项目/产品线
- 技术领域、发明内容概述、创新点
- 专利性评估（新颖性、创造性、实用性）
- 商业价值评估（市场规模、竞品情况、战略意义）
- 评审意见、评审结论、决策结果

#### 阶段2：申请期

| 子阶段 | 说明 | 关键节点 | 法定期限 |
|--------|------|---------|---------|
| 委托代理 | 选择代理机构/代理人 | 委托合同 | - |
| 申请文件撰写 | 说明书、权利要求、摘要 | 申请文件初稿 | - |
| 内部审核 | 公司内部审核申请文件 | 审核意见 | - |
| 提交申请 | 向专利局提交申请 | 申请日、申请号 | - |
| 受理 | 收到受理通知书 | 受理通知书 | - |
| 初审 | 形式审查 | 初审合格 | - |
| 公开 | 申请公开 | 公开日、公开号 | 申请日起18个月 |

**核心字段**：
- 申请号、申请日、公开号、公开日
- 申请人、发明人、代理机构、代理人
- 优先权信息（优先权号、优先权日、优先权国家）
- 申请类型（发明、实用新型、外观设计）
- 申请国家/地区

#### 阶段3：审查期

| 子阶段 | 说明 | 关键节点 | 典型期限 |
|--------|------|---------|---------|
| 实审请求 | 提出实质审查请求 | 实审生效 | 申请日起3年内（中国） |
| 一通 | 第一次审查意见通知书 | 发文日 | - |
| 答复一通 | 答复审查意见 | 答复日 | 4个月（中国） |
| 二通/三通... | 后续审查意见 | 各次通知书 | - |
| 授权 | 授权通知 | 授权日 | - |
| 驳回 | 驳回决定 | 驳回日 | - |

**核心字段**：
- 审查意见历史（每次通知书的日期、内容、答复期限）
- 答复历史（每次答复的日期、内容、修改说明）
- 当前审查状态
- 下次答复期限

#### 阶段4：授权期/注册期

| 子阶段 | 说明 | 关键节点 |
|--------|------|---------|
| 授权公告 | 专利授权公告 | 授权公告日、专利号 |
| 办登缴费 | 办理登记手续、缴纳费用 | 办登日 |
| 专利生效 | 专利权生效 | 生效日 |

#### 阶段5：维护期

| 子阶段 | 说明 | 持续时间 |
|--------|------|---------|
| 年费缴纳 | 每年缴纳年费维持专利权 | 保护期内 |
| 权利状态监控 | 监控法律状态变化 | 保护期内 |
| 专利运用 | 许可、转让、质押、维权 | 保护期内 |

#### 阶段6：终态

| 终态类型 | 说明 |
|---------|------|
| 专利权终止 | 未缴年费、保护期届满 |
| 专利权无效 | 被无效宣告 |
| 专利放弃 | 主动放弃 |
| 驳回失效 | 申请被驳回，未复审 |
| 撤回 | 主动撤回申请 |
| 视为撤回 | 未按规定答复视为撤回 |

### 2.3 状态机设计

**注意**：不同国家、不同类型的专利，状态流转规则不同。
→ 设计为**可配置的状态机**，每种专利类型+国家有自己的状态图。

## 三、跨国专利与同族管理

### 3.1 核心概念

| 概念 | 定义 | 管理要点 |
|------|------|---------|
| **优先权** | 首次申请后，在一定期限内就相同主题在他国申请，可享受首次申请日的待遇 | 优先权号、优先权日、优先权国家、优先权类型（巴黎公约/PCT） |
| **PCT国际申请** | 通过一个国际申请，指定多个国家，30/31个月后进入国家阶段 | PCT申请号、国际申请日、国际公布号、进入国家阶段期限 |
| **同族专利** | 基于同一优先权的一组专利，保护相同的发明创造 | 族编号、成员列表、优先权链 |
| **分案申请** | 从原申请中分出的新申请，保留原申请日 | 原申请号、分案原因 |
| **继续申请** | 美国特有，基于原申请的继续申请 | 原申请、继续类型（Continuation/CIP/Divisional） |

### 3.2 同族关系模型

```
同族 (PatentFamily)
├── 族编号 (family_id)
├── 族类型
│   ├── 简单同族（同一优先权）
│   ├── 复杂同族（多个优先权，有重叠）
│   ├── 延伸同族（包括引用/被引）
│   └── 专利家族（同一申请人的相关专利）
├── 优先权链 (PriorityChain)
│   └── 按时间排序的优先权关系
└── 成员专利列表
    ├── 每个成员的国家、申请号、申请日
    ├── 成员类型（基础申请、等同、改进...）
    └── 当前法律状态
```

### 3.3 优先权链可视化

时间线形式展示优先权关系：

```
优先权1 (CN, 2023-01-15)
    │
    ├── 中国申请 (CN20231012345, 2023-01-15) ──→ 授权
    │
    ├── PCT申请 (PCT/CN2024/071234, 2024-01-12)
    │       │
    │       ├── 美国国家阶段 (US18/234,567, 2025-07-15) ──→ 实审中
    │       ├── 欧洲专利 (EP24789012, 2025-07-15) ──→ 实审中
    │       └── 日本申请 (JP2025-123456, 2025-07-15) ──→ 审查中
    │
    └── 分案申请 (CN20241023456, 2024-03-20) ──→ 初审中
```

### 3.4 国家阶段进入期限管理

| 途径 | 进入期限 | 宽限期 | 错过后果 |
|------|---------|--------|---------|
| PCT - 第I章 | 自优先权日起30个月 | 部分国家可延期1-2个月 | 丧失进入权利 |
| PCT - 第II章 | 自优先权日起31个月 | - | 丧失进入权利 |
| 巴黎公约 | 自优先权日起12个月 | 部分国家有恢复程序 | 丧失优先权 |

**重要**：这是专利部门最核心的期限之一，错过就进不去了，必须有严格的提醒和升级机制。

## 四、不同技术领域的专利特性

### 4.1 技术领域分类

| 大类 | 子类示例 | 专利特点 |
|------|---------|---------|
| **机械领域** | 机械结构、制造工艺、自动化设备 | 附图多、权利要求偏结构、审查周期中等 |
| **电子/电气** | 电路、半导体、通信、传感器 | 技术更新快、同族多、国际化程度高 |
| **软件/算法** | 算法、AI、软件方法、数据处理 | 客体问题、撰写难度大、审查标准变化快 |
| **化工/材料** | 新材料、化学配方、制药 | 实验数据多、保护期长、专利布局复杂 |
| **生物/医药** | 生物序列、药物配方、医疗器械 | 监管严格、试验周期长、专利价值高 |
| **外观设计** | 产品外观、GUI、包装 | 审查快、保护期短、直观性强 |

### 4.2 不同领域的字段差异

| 领域 | 特有字段 | 说明 |
|------|---------|------|
| **通用** | 申请号、申请日、申请人、发明人、标题、摘要、权利要求... | 所有专利都有的基础字段 |
| **软件/算法** | 算法类型、应用场景、数据流程图、模型架构图 | 软件专利的特殊关注点 |
| **机械** | 技术问题、技术方案、有益效果、附图说明 | 机械专利的撰写重点 |
| **化工/材料** | 原料配比、制备方法、性能数据、实施例 | 化工专利的核心数据 |
| **生物/医药** | 序列表、实验数据、临床试验阶段、适应症 | 医药专利的特殊内容 |
| **外观设计** | 设计要点、产品用途、六面视图、使用状态图 | 外观设计的特殊内容 |

### 4.3 设计原则：字段可配置，不硬编码

**不要**为每个技术领域单独建表或硬编码字段。
**要**通过自定义字段系统 + 字段模板来实现：

1. **字段模板**：
   - 预设"软件专利模板"、"机械专利模板"等
   - 用户选择模板后，自动创建该领域常用的自定义字段
2. **字段分组**：
   - 字段按"技术领域"分组显示
   - 用户可以自己调整哪些字段显示、哪些隐藏
3. **可扩展**：
   - 用户随时可以新增/修改/删除自定义字段
   - 不受技术领域限制

## 五、专利与其他实体的关系

### 5.1 关系全景

```
专利 (Patent)
  │
  ├── 引用关系 → 其他专利（引证/被引）
  ├── 同族关系 → 同族专利
  ├── 项目关联 → 项目（多对多，有角色）
  ├── 标签关联 → 标签（多对多）
  ├── 人员关联 → 人员（发明人/负责人/代理人...）
  ├── 产品关联 → 产品线/产品（多对多）
  ├── 风险关联 → 风险事件（风险专利、诉讼、无效）
  ├── 文档关联 → 文档（申请文件、审查意见、答复意见...）
  ├── 费用关联 → 费用记录（官费、代理费、年费...）
  ├── 期限关联 → 期限（Docketing）
  └── 双向链接 → Wiki页面（[[链接]]）
```

### 5.2 项目关联（核心）

**一个专利可以属于多个项目，一个项目有多个专利，且在不同项目中角色不同。**

| 关联角色 | 说明 | 典型场景 |
|---------|------|---------|
| **核心专利** | 项目的核心技术，保护产品关键功能 | 自研技术的专利布局 |
| **改进专利** | 在核心专利基础上的改进/延伸 | 迭代优化的专利 |
| **外围专利** | 围绕核心技术的外围布局 | 专利组合、围栏布局 |
| **防御专利** | 为了防止竞争对手申请而申请 | 防御性公开、储备专利 |
| **风险专利** | 竞品的专利，可能对我方构成风险 | 风险排查、FTO |
| **参考专利** | 技术调研中的参考文件 | 技术研究、现有技术参考 |
| **竞品专利** | 竞争对手的专利，用于监控 | 竞品分析、专利监控 |
| **标准专利** | 涉及标准的专利 | 标准必要专利 |

**每个关联关系还有**：
- 关联日期、关联人
- 关联说明/备注
- 重要度评级
- 应对措施（如果是风险专利）
- 负责人

### 5.3 产品线关联

**为什么产品线和项目要分开？**
- 项目是临时性的（有起止时间）
- 产品线是持续性的（长期存在）
- 一个产品线下面有多个项目/产品版本

```
产品线 (ProductLine)
  └── 项目 (Project)
       └── 专利 (Patent)
```

## 六、Docketing（期限管理）模型

### 6.1 期限类型

| 类别 | 示例 | 重要性 |
|------|------|--------|
| **法定期限** | 优先权期限、PCT国家阶段进入期限、审查意见答复期限、年费期限 | ⭐⭐⭐⭐⭐ |
| **指定期限** | 审查员指定的期限、补正期限 | ⭐⭐⭐⭐ |
| **约定期限** | 与代理机构约定的期限、内部期限 | ⭐⭐⭐ |
| **内部期限** | 内部评审期限、提案期限 | ⭐⭐⭐ |
| **提醒期限** | 自定义提醒，如技术调研截止日 | ⭐⭐ |

### 6.2 期限状态

```
未到期 → 即将到期（黄色预警）→ 紧急（红色预警）→ 已完成 → 已逾期
```

### 6.3 提醒规则

| 提醒级别 | 触发条件 | 通知方式 | 接收人 |
|---------|---------|---------|--------|
| **一级提醒** | 到期前30天 | 弹窗+邮件 | 责任人 |
| **二级提醒** | 到期前15天 | 弹窗+邮件+桌面通知 | 责任人 + 直接主管 |
| **三级提醒** | 到期前7天 | 弹窗+邮件+桌面通知+短信 | 责任人 + 部门负责人 |
| **紧急提醒** | 到期前1天 | 所有方式+升级 | 全部相关人员 |
| **逾期提醒** | 已逾期 | 持续提醒+升级 | 全部+更高层 |

### 6.4 期限自动计算规则

**可配置的规则引擎**：
- 基于触发事件（申请日、优先权日、收到通知书日...）
- 基于法定期间（X个月、Y天）
- 考虑节假日顺延
- 考虑各国差异（中国、美国、欧洲、日本...）

**示例规则**：
```
规则名称：中国发明专利第一次审查意见答复期限
触发事件：收到第一次审查意见通知书
计算方式：发文日 + 4个月
顺延规则：遇节假日顺延到下一个工作日
提醒级别：三级提醒（30天/15天/7天）
责任人：当前专利的负责人
```

## 七、专利质量与分级模型

### 7.1 分级维度

| 维度 | 评估指标 | 权重建议 |
|------|---------|---------|
| **技术价值** | 创新高度、技术难度、技术通用性 | 30% |
| **商业价值** | 市场规模、产品相关性、竞争态势、收益预期 | 40% |
| **法律价值** | 权利要求稳定性、保护范围宽窄、可规避性 | 20% |
| **战略价值** | 布局意义、威慑作用、交叉许可筹码 | 10% |

### 7.2 分级方式

| 级别 | 说明 | 占比参考 | 管理策略 |
|------|------|---------|---------|
| **S级（核心）** | 最重要的核心专利 | 5% | 重点维护、全球布局、高成本投入 |
| **A级（重要）** | 重要的专利 | 15% | 正常维护、重点国家布局 |
| **B级（一般）** | 普通专利 | 50% | 选择性维护、成本控制 |
| **C级（次要）** | 价值较低的专利 | 20% | 考虑放弃、节省成本 |
| **D级（待弃）** | 基本没有价值 | 10% | 放弃或低价转让 |

### 7.3 评估机制

- **人工评定**：专利工程师 + 技术专家 + 产品经理共同评估
- **AI辅助评分**：基于专利文本、引用情况、同族情况等自动给出参考评分
- **定期复评**：每年/每半年重新评估一次，动态调整级别
- **评审记录**：每次评估都有记录，可追溯

## 八、费用与成本管理模型

### 8.1 费用类型

| 大类 | 明细 | 发生阶段 |
|------|------|---------|
| **申请阶段费用** | 申请费、优先权要求费、实审费、印刷费、印花税 | 申请期 |
| **代理费用** | 申请代理费、检索代理费、审查意见答复代理费 | 申请/审查期 |
| **翻译费用** | 说明书翻译、权利要求翻译、审查意见翻译 | 跨国申请 |
| **年费** | 各年度年费（逐年递增） | 维护期 |
| **年费滞纳金** | 逾期缴费的滞纳金 | 维护期 |
| **无效/诉讼费用** | 无效请求费、诉讼费、律师费 | 纠纷期 |
| **其他费用** | 检索费、公证费、认证费、差旅费 | 各阶段 |

### 8.2 费用管理功能

1. **费用记录**：
   - 录入每一笔费用（类型、金额、日期、支付状态）
   - 关联到具体专利、具体阶段
2. **费用预测**：
   - 自动预测未来N年的年费支出
   - 按部门/项目/产品线统计
3. **预算管理**：
   - 年度预算设置
   - 预算执行情况跟踪
4. **费用分析**：
   - 单件专利生命周期总成本
   - 各类型费用占比
   - 不同国家费用对比
5. **缴费提醒**：
   - 年费缴纳提醒
   - 与Docketing集成

## 十一、MVP业务范围建议

### 11.1 MVP 做什么

**聚焦核心，先把基础打牢**：

1. **专利基础数据管理**
   - 核心字段（约30-50个）
   - 自定义字段系统（基础版）
   - 增删改查

2. **表格视图（类Excel）**
   - AG Grid实现
   - 筛选、排序、分组
   - 批量编辑

3. **Excel导入导出**
   - 字段映射
   - 智能去重合并
   - 导入预览

4. **单专利详情页**
   - 基础信息展示
   - 自定义字段面板
   - Wiki正文（Markdown）

5. **标签系统**
   - 标签增删改
   - 专利打标签
   - 按标签筛选

6. **项目关联**
   - 项目管理
   - 专利-项目关联（带角色）
   - 项目维度查看专利

### 11.2 MVP 不做什么（第二期再做）

- 审查流程管理、状态机
- Docketing期限管理
- 费用管理
- 同族管理（先做简单的，复杂的第二期）
- 引用图谱（先做列表，图谱第二期）
- AI功能（基础版可以有，但不是核心）
- 用户权限（单用户先不做权限）
- 工作流/审批
